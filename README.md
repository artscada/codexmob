# codex-android-RU

Русифицированный и доработанный форк `codex-android` для Android-устройств с упором на локальный Codex, Linux runtime на `chroot`, root-сценарии и HTTP API для внешней оркестрации.

Этот репозиторий собран так, чтобы можно было:

- скачать APK из релиза;
- установить его на телефон без сборки проекта;
- настроить локальный или внешний LLM-провайдер;
- включить HTTP API на телефоне;
- проверить работу устройства готовым тестовым скриптом без помощи из чата.

## Что добавлено в этом форке

- Русификация интерфейса: добавлен пакет `values-ru`, переведены основные экраны и настройки.
- История чатов: чаты и сообщения сохраняются локально, можно создавать новые диалоги и возвращаться к старым.
- Локальный Codex CLI: добавлены настройки провайдера для локального режима, синхронизация конфигурации и повторный запуск runtime.
- Linux runtime направление: `proot` остается как fallback, а `chroot` становится целевым основным режимом для полноценного Python и скриптов.
- Внешний HTTP API: реализованы `health`, `run-task`, журнал запусков, `cancel`, потоковый SSE и `async_callback`.
- Журнал задач: история HTTP/API запусков хранится отдельно от истории чата.
- Поддержка обратного callback по локальной сети: разрешён cleartext HTTP для LAN и добавлены примеры настройки.
- Root / shell сценарии: улучшена прямая обработка явных shell/root-команд и политика безопасности.
- Улучшения диагностики: обновлены диагностические экраны и проверки окружения.

## Где взять APK

После публикации релиза APK можно взять во вкладке GitHub Releases.

При локальной подготовке артефакты создаются здесь:

- `artifacts/release/codex-android-ru-207-v1.11.0_7.apk`
- `artifacts/release/SHA256SUMS.txt`

Текущий опубликованный артефакт в этом репозитории основан на рабочей сборке,
снятой с устройства `207` как `latest known good`.

## Быстрый старт

1. Скачай APK из релиза.
2. При желании сверь SHA256 из `SHA256SUMS.txt`.
3. Установи его на телефон.
3. В приложении настрой провайдера модели.
4. Включи внешний HTTP API и сохрани токен.
5. Запусти тестовый скрипт из `tools/test_http_api_device.ps1`.

## Важные ограничения

- Длинные свободные диагностические запросы в локальном Codex могут выполняться дольше ожидаемого или подвисать. Для инженерных сценариев лучше использовать жёсткие структурированные prompt-шаблоны и HTTP/API журнал задач.
- Русификация ещё не полная: часть строк пока остаётся на английском.
- В релиз не включаются приватные ключи, токены, пользовательские скриншоты и временные отладочные логи.

## Сборка из исходников

```powershell
./gradlew.bat assembleDebug
```

Готовый APK будет в:

```text
app/build/outputs/apk/debug/codex-android-debug.apk
```

## Самопроверка HTTP API

Универсальный Windows-скрипт лежит здесь:

- `tools/test_http_api_device.ps1`

Он умеет проверить:

- `GET /api/health`
- `POST /api/run-task` в режиме `sync`
- root-диагностику
- `stream=true` через SSE
- `async_callback`
- список задач и `cancel`

Пример запуска:

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\test_http_api_device.ps1 `
  -BaseUrl "http://192.168.28.207:8094" `
  -Token "YOUR_BEARER_TOKEN" `
  -CallbackHost "192.168.28.230"
```

## Для публикации релиза

В репозитории уже предусмотрена структура для самостоятельной публикации:

- код приложения;
- русская документация;
- готовый релизный APK;
- контрольная сумма;
- тестовый сценарий для HTTP API.

Если GitHub CLI уже авторизован:

```powershell
gh release create v2026.06.02-ru.1 `
  artifacts/release/codex-android-ru-207-v1.11.0_7.apk `
  artifacts/release/SHA256SUMS.txt `
  --title "codex-android-RU 207 latest known good (v1.11.0+7)" `
  --notes-file docs/RU_RELEASE_GUIDE.md
```
